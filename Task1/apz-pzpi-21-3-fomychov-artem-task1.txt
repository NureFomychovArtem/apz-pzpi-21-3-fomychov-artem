1.1.	Background (Передумови)

Останнім часом, більшість навчальних закладів перейшла на дистанційний формат навчання, але згодом всі школи будуть працювати як це було до 2020 року. Діти так само будуть відвідувати заклади, формувати гурти та займатися своїми захопленнями. Для цього потрібно, щоб школа була готова до нових ідей та нового часу, потрібно щоб дітям та працівникам було зручно займатися своєю справою і щоб вони робили це із задоволенням, потрібно щоб була комунікація між школярами та викладачами та не було потреби шукати один одного по всьому навчальному закладі. 
Проект «EduCheck» пропонує інноваційний підхід для комунікації між відвідувачами закладів. Головна ідея полягає в створенні зручної та надійної системи, яка буде містити інформацію про відвідувачів (дату та час відвідування школи, температуру, пульс та тиск), надає змогу комунікації між користувачами системи, бронювання загальнодоступних кімнат, залів та класів для зайнять Наш продукт дасть можливість користувачам здійснювати більшого функціоналу в декілька рухів у мобільному додатку. 

1.2.	Business Opportunity (Бізнес-можливості)

Більшість з закладів не мають жодних систем для відстежування відвідування закладів, систем комунікації та інформування стосовно зайнятості залів, класів тощо. Недоліками таких закладів є:
-	Відсутність системи безпеки,
-	Погіршена система комунікації між школярами та викладачами,
-	Відсутність відстежування зайнятості приміщень,
-	Загальне введення статистики відвідування.
Наш продукт орієнтується на заклади освіти, а саме школи. 
Ми плануємо забезпечити безкоштовний доступ користувачам до нашої системи. Наша прибуткова модель полягає у продажі системи за собівартістю, фінансування благодійними організаціями та державою, проведення реклами погодженою з керівництвом, так як основна мета проекту – це забезпечити зручне навчання.

2.1.	Vision Statement (Окреслення концепції)

Система «EduCheck» надаватиме послуги зі збору інформації про відвідувачів закладу, комунікації між школярами та працівниками закладу, створення запитів та інформування. Обрати та забронювати приміщення можна в мобільному застосунку користувача, попередньо обравши дату, час та мету перебування. Також надано функціонал для с.
Працівники закладу можуть відстежувати відвідування учнів, їх стан здоров’я та їх зайнятість. Також буде надана можливість перегляду загальної статистики, формування оголошень та запитів на бронювання приміщення або погодження подій.

2.2.	Major Function (Головна функціональність)

MF-1: Сканування часу прибуття до закладу та температури за допомогою сканеру відбитку пальця.
MF-2: Створення оголошень стосовно змін.
MF-3: Створення та обробка запитів на бронювання приміщення або проведення подій.
MF-4: Перегляд та редагування розкладу занять.
MF-5: Перегляд статистики зайнятості школярів.
MF-6: Надання системою безпеки персональних даних.
MF-7: Можливість обрати українську або англійську локалізацію інтерфейсу програмної системи.

3.1 Scope and Initial Release (Рамки первинного випуску)

У першому запуску продукту будуть реалізовані лише основні функції для успішної роботи системи, включаючи адміністрування, локалізацію, захист персональних даних. 
Серверна частина (Back-end):
-	Створення облікових записів для користувачів (відвідувачів та адміністраторів закладу).
-	Система авторизації та автентифікації для безпечного доступу до системи(використання jwt-tokens).
-	Зберігання інформації про відвідування, зміни та запити на сервері.
-	Створення запитів на бронювання приміщень або проведення подій.
-	Система для запитів(додавання/редагування/обробка запитів у закладі).
-	Комунікація між школярами та викладачами. 
Клієнтська частина (Front-end):
-	Локалізація веб-інтерфейсу.
-	Веб-інтерфейс для закладів для моніторингу інформації, обробки та перегляду запитів. .
-	Веб-інтерфейс для редагування розкладу.
-	Відображення аналітичних даних про відвідування та стан школярів.
Мобільний програмний застосунок:
-	Локалізація інтерфейсу додатку.
-	Перегляд інформації про себе.
-	Створення запиту на проведення події або бронювання класу, залу.
-	Перегляд розкладу занять та змін до нього.
-	Можливість відмітитись самостійно. 
IoT програмний застосунок:
-	Отримання сигналу про вхід/вихід особи.
-	Сканування відбитку пальця
-	Сканування температури тіла.

3.2 Scope of Subsequent Release (Рамки наступних випусків)

1.	Додавання електронних щоденників та системи оцінювання;
2.	Додавання системи додаткових балів для школярів та працівників;
3.	Оформлення підписки для додаткового функціонала для закладів;
4.	Створення звітів у форматі таблиць Excel про відвідувачів за певні проміжки часу;
5.	Створення телеграм-боту для інформування стосовно розкладу;
6.	Інтеграція ШІ для аналізу та обробки даних. 

4.3 Operation Environment (Робоче середовище)
Програмна система складається з серверної частини на мові C# (платформа .NET Core 8, ASP.NET Core, REST API). В якості бази даних використовується SQL Server, СУБД - MS SSMS. Веб-клієнт виконаний за допомогою Angular Framework. Мобільний застосунок використовує технологію .NET MAUI. Для IoT планується використовувати симулятор Wokwi.
Даний стек технологій є кросплатформений, тому сервісом можна буде користуватися на будь-яких пристроях.
